# Ride-Hailing-Demand-and-Pricing-Analysis
This project applies Big Data analytics techniques to analyse ride-hailing data from Uber and Lyft in Boston, Massachusetts. The objective is to understand how factors such as time of day, trip distance, weather conditions, and surge pricing influence ride demand and fare prices.
The project combines Python for data cleaning and exploratory data analysis (EDA) with Power BI for dashboard creation and business intelligence reporting.

#Business Problem
Ride-hailing companies operate in highly dynamic environments where demand changes throughout the day. These fluctuations can result in:

Driver shortages during peak periods
Increased passenger waiting times
Dynamic surge pricing
Revenue and resource allocation challenges

This analysis aims to identify the key factors influencing demand and pricing to support data-driven decision-making.

#Tools Used
1.Python (Pandas, NumPy, Matplotlib, Seaborn)
2.Microsoft Power BI
3.Jupyter Notebook

#Data Preperations
The dataset was cleaned and transformed using Python. Key feature engineering tasks included:

1.Creating time period categories (Morning, Afternoon, Evening, Night)
2.Classifying rides as Weekday or Weekend
3.Grouping trips into distance categories
4.Preparing weather-related variables for analysis

#Analysis Performed
1.Exploratory Data Analysis (EDA)
2.Demand patterns by time period
3.Fare distribution analysis
4.Weather impact on pricing
5.Company performance comparison

Correlation Analysis
The strongest factors influencing fare price were Distance and Surge Multiplier

#Key Findings
*Ride demand varies significantly throughout the day.
*Distance is the strongest predictor of fare price.
*Weather conditions can contribute to fare increases.
*Uber and Lyft exhibit similar demand trends.
*Data analytics can support pricing optimisation and demand forecasting.
